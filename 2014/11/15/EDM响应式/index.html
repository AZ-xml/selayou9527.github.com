客户端支持

  1. 响应式本身只有IOS和OS X 的mail客户端支持
  2. Gmail、Yahoo!、outllok.com 不支持引用的外联样式
  3. Gmail及Android Mail客户端对style标签head、body内的都会删除
  4. QQ邮箱会把head中的style标签保存到body，然后删除body外内容，但是同时会删除media query的style内容

响应式涉及样式

选择器

  1. Gmail因为会删除style标签，只支持内联的style样式，所有所有选择器都不支持，不单独指出
  2. E 全部支持
  3. E[foo] 属性选择器outlook和Yahoo!基本不支持
  4. E:foo 伪类、伪元素选择器outlook和Yahoo!基本不支持，a标签的伪类Yahoo!支持
  5. .class 类选择器全部支持
  6. #id id选择器outlook.com不支持
  7. E F 后代选择器全部支持
  8. E>F 子元素选择器或类似的兄弟选择器outlook不支持

属性

  1. direction 全部支持
  2. font相关全部支持，字体大小IOS限制最小为13px
  3. line-height outlook.com不能为td设置
  4. text-align、text-decoration、color全部支持
  5. border全部支持
  6. height outlook不支持
  7. width outlook不支持P、DIV的设置
  8. margin outlook.com不支持
  9. padding outlook不支持P、DIV的设置
  10. max-width、min-width outlook不支持
  11. position、display、float、left 等位置移动属性outlook不支持

基本思路

看完上面支持情况，不禁倒吸一口凉气，这也能实现响应式？Gmail上几乎是一个不能完成的任务！

只要开动脑筋，其实我们是可以做到邮箱的响应式的

比网页好的地方在于因为大部分邮件客户端时outlook，outlook默认的邮件内容可视区域并不大，所以大部分邮件的宽度都控制在700px左右

这就意味着我们不需要考虑PC网页及客户端的响应式，只需要考虑Apple Mail和Android Mail等主流邮件客户端的响应式

其中Apple Mail使用Webkit内核，基本使用网页的响应式就可以了，我们把精力用在Android Mail上即可

  1. table的align＝"left"可以让table实现类似float:left的效果，width靠内容撑开
  2. Gmail虽然不支持style标签，但是支持内联的max-width、min-width

开搞！

PC显示700px，移动端响应式

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">
<head>
  <meta http-equiv="Content-Type" content="text/html; charset=utf-8" />
  <meta name="viewport" content="width=device-width"/>
  <meta name="format-detection" content="telephone=no,adress=no"/>
    <style type="text/css">
      #outlook a { padding:0; }
      body{ width:100% !important; min-width: 100%; -webkit-text-size-adjust:100%; -ms-text-size-adjust:100%; margin:0; padding:0;}
      .ExternalClass { width:100%;}
      .ExternalClass, .ExternalClass p, .ExternalClass span, .ExternalClass font, .ExternalClass td, .ExternalClass div { line-height: 100%; }
      #backgroundTable { margin:0; padding:0; width:100% !important; line-height: 100% !important; }
      table{border-collapse:collapse; mso-table-lspace:0pt !important; mso-table-rspace:0pt !important;}
      td{ border-collapse: collapse; word-break:break-all; }
      img { outline:none; text-decoration:none; -ms-interpolation-mode: bicubic;width: auto;max-width: 100%;}
      p { margin: 0px 0px !important;}
      .edm-wrap{ width:700px !important;}

      @media only screen and (min-width:720px){
        .edm-wrap{ width:auto !important; }
      }
      @media only screen and (max-width: 480px) {
        .edm-wrap{ width:320px !important; }
        .edm-container{padding:8px;}
      }
    </style>
<body>
    <table bgcolor="#efefef" cellpadding="0" cellspacing="0" border="0" width="100%">
      <tr>
        <td align="center" style="padding-top:30px;padding-bottom:30px;">
          <table class="edm-wrap" align="center" cellpadding="0" cellspacing="0" border="0" style="min-width:320px;max-width:700px;width:auto;">
            <tr>
              <td id="content" class="edm-container" bgcolor="#ffffff" style="padding-left:20px;padding-right:20px;padding-bottom:20px;border-left:solid 1px #cccccc;border-right:solid 1px #cccccc;border-bottom:solid 1px #cccccc;">
                <!-- write content here -->
              </td>
            </tr>
          </table>
        </td>
      </tr>
    </table>
</body>
</html>

这就是主体部分，上半部分style，对于支持响应式的自然没有问题，对于Gmail主要靠内敛样式

body最外层的table控制内容居中对其，class="edm-wrap"是邮件内容真正的容器，为其添加内联样式

min-width:320px;max-width:700px;width:auto;

而上面的style中还有针对它的样式

.edm-wrap{ width:700px !important;}

@media only screen and (min-width:720px){
    .edm-wrap{ width:auto !important; }
}
@media only screen and (max-width: 480px) {
    .edm-wrap{ width:320px !important; }
    .edm-container{padding:8px;}
}

PS. !important 优先级比内联样式高

  1. 对于Apple Mail由于支持响应式会在720及以上按照max-width:700px，480及以下使用width:320px;
  2. 对于QQ邮箱和Outlook等只支持普通style的会使用width:700px
  3. 对于Gmail等不支持外部style的会使用内联样式min-width:320px;max-width:700px;width:auto;

内容

这样我们控制了容器在700px～320px之间变化，具体宽度的就是靠内容撑开了，我们需要利用table的align属性，让table变成float的

<!-- 不要声明table的宽度，靠内容决定，这样在320下后面的table会自动下沉 -->
<table align="left" cellpadding="0" cellspacing="0" border="0">
    <tr>
        <td></td>
    </tr>
</table>

最后

靠上面的两个建议就基本上实现了简单的响应式

  1. 外层通过外部style的!important、Media Query、内联样式的max-width与min-width配合使用，使容器宽度控制在700px和320px
  2. 内容部分使用table的align属性使table“浮动”

当然有个流行的EDM版的bootstrap ink框架，效果在部分邮箱很不错，其内容使用的是!important、Media Query所以对于Gmail肯定是不作用的，有兴趣的同学可以看看，本文的很多思路与style都是借鉴的ink

大漠有篇不错的文章专门介绍 Ink框架快速创建响应式EDM模板
