find命令的xargs

在Linux上find命令的曝光率很高，我们希望对find查找出的文件进行过滤或者操作怎么办呢，这时候就可以利用xargs了，它可以把find结果当作下一个指令的参数传入

find . -name * | xargs grep '.'

查找文件夹下包含. 的文件

find . -name *.txt | xargs rm -rf

删除所有.txt文件

scp

scp就是secure copy，在linux下用来进行远程拷贝文件/文件夹

scp -p port remoteUser@host:/path localPath

pgrep

grep命令我们都很熟悉，查找进程我们可以使用pgrep pgrep 通过程序的名字来查询进程的工具，一般是用来判断程序是否正在运行

常用参数

-l 列出程序名和进程ID；
-o 进程起始的ID；
-n 进程终止的ID；

｀pgrep -l nginx'

查看机器运行的 nginx服务进程

uname

uname -a 显示机器内核等信息

查找

find

这个时大家常用的

find <指定目录> <指定条件> <指定动作>

 find . -name '*.php' -ls

locate

Linux中有一个包含所有本地文件信息的数据库(/var/lib/locatedb)，这个数据库每天更细一次，所以结果不是实时的，但是我们可以通过updatedb命令手工更新数据库

locate命令正式查找这个数据库的信息来定位文件，locate相当于find -name的快速版

locate -i ~/a

搜索当前文件夹下 a 开头的文件，忽略大小写

whereis

whereis命令用于程序名的搜索

    whereis grep

which

which命令的作用是，在PATH变量指定的路径中，搜索某个系统命令的位置，并且返回第一个搜索结果。也就是说，使用which命令，就可以看到某个系统命令是否存在，以及执行的到底是哪一个位置的命令

which grep

bc

进入数学计算

lsof -i

lsof -i 80

查看端口号被哪个进程占用

set

显示系统变量（env可以显示一部分）
