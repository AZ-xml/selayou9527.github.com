环境

在开始写代码之前我们需要了解Go的运行环境

Workspaces

Go的代码必须在一个workspace中，workspace的目录结构

  • src 包含组织到package的Go代码文件，一个package对应一个文件夹
  • pkg 包含各种package
  • bin 可运行文件

看个例子

bin/
    streak                         # command executable
    todo                           # command executable
pkg/
    linux_amd64/
        code.google.com/p/goauth2/
            oauth.a                # package object
        github.com/nf/todo/
            task.a                 # package object
src/
    code.google.com/p/goauth2/
        .hg/                       # mercurial repository metadata
        oauth/
            oauth.go               # package source
            oauth_test.go          # test source
    github.com/nf/
        streak/
            .git/                  # git repository metadata
            oauth.go               # command source
            streak.go              # command source
        todo/
            .git/                  # git repository metadata
            task/
                task.go            # package source
            todo.go                # command source

GOPATH

和Java、python等一样Go也需要配置环境变量，其中GOPATH不是Go的bin目录，而是workspace目录。这也是Go需要的唯一的环境变量，workspace可以设置到任何非GO安装路径的地方

$ mkdir $HOME/go
$ export GOPATH=$HOME/go

为了方便编译后的可执行文件运行，可以把workspace的bin目录配置到PATH中

$ export PATH=$PATH:$GOPATH/bin

package路径

系统package都有了比较短的名字fmt、net/http等，自定义的package自己要组织好路径，像Java一样使用域名活着自己github的名字等

Hello World!

一切都准备妥当了，终于可以开始写Hello World!了，在src目录下创建合适目录结构

$ mkdir $GOPATH/src/github.com/user/hello

创建hello.go测试文件

package main

import "fmt"

func main() {
    fmt.Printf("Hello, world.\n")
}

用过Java的同学一看就知道什么意思，编译执行一下文件

$ go install github.com/user/hello

这句命令可以在任何地方运行，Go会根据GOPATH查找源文件，当然也可以进代码目录后编译

$ cd $GOPATH/src/github.com/user/hello
$ go install

这句命令在workspace的bin目录下编译出一个可执行文件，只有在遇到错误的时候终端才会有显示，没显示就说明编译成功，这样就能够运行了

$ $GOPATH/bin/hello
Hello, world.

如果把workspace的bin目录添加到PATH变量中了，就更方便了

$ hello
Hello, world.

编写linrary

所谓library就是库函数，供程序复用。第一步同样是创建一个package

$ mkdir $GOPATH/src/github.com/user/newmath

然后创建 sqrt.go 文件，语法先不计较

// Package newmath is a trivial example package.
package newmath

// Sqrt returns an approximation to the square root of x.
func Sqrt(x float64) float64 {
    z := 1.0
    for i := 0; i < 1000; i++ {
        z -= (z*z - x) / (2 * z)
    }
    return z
}

单独编译一下这个文件

$ go build github.com/user/newmath

如果就在当前目录下

$ go build

这样不会有任何文件生成，需要使用install命令，这样会把package对象放到pkg文件夹内

这时候可以稍微修改一下hello.go

package main

import (
    "fmt"

    "github.com/user/newmath"
)

func main() {
    fmt.Printf("Hello, world.  Sqrt(2) = %v\n", newmath.Sqrt(2))
}

Go会自动处理依赖关系，所以

$ go install github.com/user/hello

这样newmath也会被安装，运行一下新版本的hello world

$ hello
Hello, world.  Sqrt(2) = 1.414213562373095

这时候目录结构会变成大概这样

bin/
    hello              # command executable
pkg/
    linux_amd64/       # this will reflect your OS and architecture
        github.com/user/
            newmath.a  # package object
src/
    github.com/user/
        hello/
            hello.go   # command source
        newmath/
            sqrt.go    # package source

pkg 文件夹内的newmath.a就是编译出的library文件

Package 名字

Go语言代码文件的第一句都是

package name

name用来表示package和被其他文件引用，在一个package文件夹下的文佳必须使用同样的package名称，可执行文件的package名称必须是main，package的名字可以重复，只要路径＋名称唯一即可，引用的时候也是岸路径＋名称引用的

测试

测试时写代码的好习惯，Go自带轻量级的测试框架，使用go test命令和testingpackage。测试文件命名用_test.go结尾，里面方法名使用TestXXX，方法签名是func (t *testing.T)，创建一个看看

测试刚才写的newmath package，创建文件$GOPATH/src/github.com/user/newmath/sqrt_test.go

package newmath

import "testing"

func TestSqrt(t *testing.T) {
    const in, out = 4, 2
    if x := Sqrt(in); x != out {
        t.Errorf("Sqrt(%v) = %v, want %v", in, x, out)
    }
}

使用go test运行测试命令

$ go test github.com/user/newmath
PASS
ok      github.com/user/newmath 0.165s

当然在package目录下的话，直接运行也可以

$ go test
PASS
ok      github.com/user/newmath 0.165s

远程package

import package的时候路径也可以写远程路径，install时Go会自动获取远程package

$ go get code.google.com/p/go.example/hello
$ $GOPATH/bin/hello
Hello, world.  Sqrt(2) = 1.414213562373095

如果package不在workspace里，Go会把它放到GOPATH配置的workspace中，用了上面的命令，目录结构变成

bin/
    hello                 # command executable
pkg/
    linux_amd64/
        code.google.com/p/go.example/
            newmath.a     # package object
        github.com/user/
            newmath.a     # package object
src/
    code.google.com/p/go.example/
        hello/
            hello.go      # command source
        newmath/
            sqrt.go       # package source
            sqrt_test.go  # test source
    github.com/user/
        hello/
            hello.go      # command source
        newmath/
            sqrt.go       # package source
            sqrt_test.go  # test source

也可以这样用

import "code.google.com/p/go.example/newmath"

使用远程package和前端使用代码托管用类似的思想
