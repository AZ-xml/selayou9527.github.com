Mac nginx PHP支持

文章目录
  1. 1. PHP-FPM
    1. 1.1. 配置
    2. 1.2. 启动
  2. 2. nginx配置

如果选用Apache做PHP web服务器配置很简单，只需要load php5_module 模块就可以了，但使用Apache也有一些缺点，Apache是多线程的，没来一个请求酒会打开一个线程处理，随着流量的增加，内存就不够用了。

即使使用Apache很多人也会选择不使用php5_module模块，而使用spawn-fcgi，因为使用php5_module模块的话Apache处理PHP的时候会load所有的库，这是一个巨大的开销。

PHP-FPM

不同于Apache，nginx是一个轻量级的服务器，专注于HTTP和反向代理，采用单线程事件模型，可以处理高并发，多进程负载均衡做的也十分出色，但是nginx本身并不支持PHP，以前很多人都是通过spawn-fcgi来使nginx支持PHP的，但PHP从5.3.3后原生的支持了PHP-FPM(PHP FastCGI Process Manager)，性能比起spawn-fcgi更加优越，所以我们使用PHP-FPM

配置

由于Mac原生安装PHP，新的版本中已经自带了PHP-FPM我们就不用安装了，配置一下PHP-FPM的配置文件在

/private/etc/php-fpm.conf

需要修改一下默认的log地址，默认的地址不存在，去掉注释;，修改一下

error_log = /usr/local/var/log/php-fpm.log

有个小设置需要注意下

daemonize＝yes

这个配置会让PHP-FPM以后台守护进程的方式打开，如果不希望这样的话可以设置为no

启动

启动很简单，直接调用命令即可，使用管理员权限

sudo php-fpm

可以把服务加到开机自启中

mkdir -p ~/Library/LaunchAgents/
cp /usr/local/Cellar/php54/5.5.3/homebrew-php.josegonzalez.php55.plist ~/Library/LaunchAgents/
launchctl load -w ~/Library/LaunchAgents/homebrew-php.josegonzalez.php55.plist

nginx配置

nginx的配置文件在

/usr/local/etc/nginx/nginx.conf

在Server节点中添加index支持

index index.html index.htm index.php;

打开Server节点下的location设置，对.php文件处理

location ~ \.php$ {
     include fastcgi.conf;
     fastcgi_pass   127.0.0.1:9000;
}

重启nginx就能解析PHP文件了
